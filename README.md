## 2. git commit提交规范

### 2.1 Commit message 格式
```text
<type>: <subject>
```
* type: 说明 commit 的类别，只允许使用下面7个标识
```text
- feat：新功能（feature）
- fix：修补bug
- docs：文档（documentation）
- style： 格式（不影响代码运行的变动）
- refactor：重构（即不是新增功能，也不是修改bug的代码变动）
- test：增加测试
- chore：构建过程或辅助工具的变动
```

* subject: 说明 commit 目的的简短描述，不超过50个字符
```text
- 以动词开头，使用第一人称现在时
- 若英文,第一个字母小写
- 结尾不加句号（.）
```
